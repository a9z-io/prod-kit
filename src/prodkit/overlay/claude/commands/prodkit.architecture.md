# /prodkit.architecture

## User Input

```text
$ARGUMENTS
```

## Goal

Scaffold `.specify/memory/architecture.md` — the canonical system architecture knowledge-base file. Agents load this for `coding` and `architecture-review` tasks. Supports `--update` mode for merging changes to specific sections without overwriting unchanged content.

## Target File

`.specify/memory/architecture.md`

## Idempotency Rule

- **Does not exist** → create using the schema below.
- **Exists, no `--update` flag** → read, identify missing `##` sections, append only missing sections. Do NOT modify existing content. Report added sections or "nothing to add."
- **Exists, `--update` flag present in `$ARGUMENTS`** → prompt the user: "Which sections have changed? (list section names)" Wait for response, then present the current content of only those sections and ask the user to provide the updated content. Merge provided updates section by section without touching any other sections.
